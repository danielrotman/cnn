#01 Fundamentals

Artificial Intelligence (AI)

The broad field of computer science focused on building systems capable of performing tasks that typically require human intelligence. AI encompasses many sub-fields including machine learning, natural language processing, and computer vision. It ranges from narrow AI that excels at specific tasks to the theoretical concept of general AI.

#02 Fundamentals

Machine Learning

A subset of AI where systems learn patterns and make decisions from data without being explicitly programmed for each scenario. Algorithms improve their performance automatically through experience by identifying statistical regularities in training data. Common paradigms include supervised, unsupervised, and reinforcement learning.

#03 Fundamentals

Deep Learning

A specialised branch of machine learning that uses neural networks with many layers (hence "deep") to model complex patterns in data. Deep learning has driven breakthroughs in image recognition, speech understanding, and generative AI. It excels when large amounts of data and computational power are available.

#04 Fundamentals

Neural Network

A computing model inspired by the structure of the human brain, composed of interconnected layers of artificial neurons. Each neuron receives inputs, applies a weighted sum and an activation function, then passes the result onward. Neural networks can approximate virtually any function given enough data and capacity.

#05 Architecture

Convolutional Neural Network (CNN)

A specialised type of neural network designed primarily for processing grid-like data such as images. CNNs use convolutional layers to automatically learn spatial hierarchies of features — from simple edges to complex objects. They are the backbone of modern computer vision systems.

#06 Architecture

Convolution

A mathematical operation that slides a small filter across the input data, computing element-wise multiplications and summing the results at each position. In CNNs, convolution extracts local features like edges, textures, and shapes from an image. Stacking convolutions builds increasingly abstract representations.

#07 Architecture

Filter (Kernel)

A small matrix of learnable weights that is convolved with input data to detect specific features. Each filter specialises in detecting a particular pattern, such as horizontal edges, colour gradients, or textures. A convolutional layer typically contains many filters to capture diverse features simultaneously.

#08 Architecture

Feature Map

The output produced when a filter is applied across an input, highlighting where specific features are detected. Each filter generates its own feature map, so a layer with 32 filters produces 32 feature maps. Deeper layers produce feature maps that encode increasingly abstract and complex patterns.

#09 Architecture

Pooling (Max Pooling)

A down-sampling operation that reduces the spatial dimensions of feature maps while retaining the most important information. Max pooling, the most common variant, selects the maximum value within each pooling window. This reduces computational cost, provides a degree of translation invariance, and helps prevent overfitting.

#10 Architecture

Stride

The number of pixels the filter moves across the input in each step during convolution or pooling. A stride of 1 means the filter shifts one pixel at a time, while a stride of 2 skips every other position. Larger strides reduce the output dimensions and computational cost.

#11 Architecture

Padding

The practice of adding extra pixels (typically zeros) around the border of an input before applying convolution. Padding preserves spatial dimensions so that the output feature map matches the input size. It also ensures that edge and corner pixels contribute equally to the learned features.

#12 Fundamentals

Neuron

The basic computational unit in a neural network, loosely modelled after biological neurons. A neuron computes a weighted sum of its inputs, adds a bias term, and passes the result through an activation function. Networks are built by connecting many neurons in layers.

#13 Activation

Activation Function

A mathematical function applied to a neuron's output to introduce non-linearity into the network. Without activation functions, even a deep network would behave like a single linear transformation. Common choices include ReLU, sigmoid, and softmax, each suited to different scenarios.

#14 Activation

ReLU (Rectified Linear Unit)

The most widely used activation function in deep learning, defined as f(x) = max(0, x). ReLU outputs zero for negative inputs and the input itself for positive values, providing a simple yet effective non-linearity. It trains faster than sigmoid or tanh because it avoids the vanishing gradient problem for positive values.

#15 Activation

Sigmoid

An activation function that squashes any real number into the range (0, 1), making it useful for binary classification outputs. Its S-shaped curve smoothly transitions between 0 and 1, providing a probabilistic interpretation. However, it suffers from the vanishing gradient problem, which limits its use in hidden layers of deep networks.

#16 Activation

Softmax

An activation function that converts a vector of raw scores (logits) into a probability distribution, where all values sum to 1. It is typically used in the final layer of a classification network to assign probabilities to each class. The class with the highest probability is chosen as the model's prediction.

#17 Training

Forward Pass

The process of passing input data through the network layer by layer, from input to output, to generate a prediction. Each layer applies its weights, biases, and activation functions sequentially to transform the data. The forward pass is the first step in both training and inference.

#18 Training

Backpropagation

The core algorithm for training neural networks, which computes how much each weight contributed to the overall error. It works by propagating the loss gradient backward through the network using the chain rule of calculus. These gradients are then used by an optimiser to update weights and improve predictions.

#19 Optimization

Gradient Descent

An iterative optimisation algorithm that adjusts model parameters in the direction that reduces the loss function most steeply. It computes the gradient (partial derivatives) of the loss with respect to each weight, then takes a step proportional to the negative gradient. It is the foundational strategy behind virtually all neural network training.

#20 Optimization

Stochastic Gradient Descent (SGD)

A variant of gradient descent that updates weights using the gradient computed from a single random sample (or small batch) rather than the entire dataset. This introduces noise that can help escape local minima and speeds up training significantly. SGD and its variants (Adam, RMSprop) are the most common optimisers in practice.

#21 Optimization

Learning Rate

A hyperparameter that controls how large each weight update step is during gradient descent. Too high a learning rate can cause the model to overshoot optimal values and diverge, while too low a rate results in painfully slow convergence. Finding the right learning rate is one of the most critical tuning decisions in training.

#22 Training

Loss Function

A function that quantifies how far the model's predictions are from the true target values, providing a single number to minimise. It guides the entire training process — backpropagation computes gradients of this function to update weights. Common loss functions include mean squared error for regression and cross-entropy for classification.

#23 Training

Cross-Entropy Loss

The standard loss function for classification tasks that measures the divergence between predicted probability distributions and true labels. It heavily penalises confident but wrong predictions, encouraging the model to produce well-calibrated probabilities. It pairs naturally with softmax outputs in multi-class classification problems.

#24 Fundamentals

Weights

The learnable numerical parameters within a neural network that determine the strength of connections between neurons. During training, weights are iteratively adjusted by the optimiser to minimise the loss function. The specific pattern of trained weights encodes everything the network has learned from the data.

#25 Fundamentals

Bias

An additional learnable parameter added to the weighted sum in each neuron before the activation function is applied. Bias allows the neuron to shift its activation threshold, enabling the network to better fit data that doesn't pass through the origin. Each neuron typically has its own independent bias value.

#26 Training

Epoch

One complete pass through the entire training dataset during the training process. Models typically require many epochs (tens to hundreds) before the weights converge to good values. Monitoring performance across epochs helps diagnose whether the model is still improving or beginning to overfit.

#27 Training

Batch

A subset of training samples processed together in a single forward and backward pass before updating the weights. Using batches instead of the full dataset makes training computationally feasible for large datasets. It also introduces beneficial noise that can improve generalisation.

#28 Training

Batch Size

The number of training samples included in a single batch for one update step. Smaller batch sizes provide noisier gradients but can improve generalisation, while larger batches give more stable updates and better GPU utilisation. Common values range from 16 to 256, depending on memory and dataset size.

#29 Data

Training Set

The portion of a dataset used to train the model by adjusting its weights through forward passes and backpropagation. It is typically the largest split, often comprising 60–80% of the total data. The quality and diversity of the training set directly determine how well the model can learn meaningful patterns.

#30 Data

Validation Set

A separate portion of the dataset used to evaluate model performance during training, without contributing to weight updates. It helps tune hyperparameters and detect overfitting by providing an unbiased estimate of model skill on unseen data. If validation accuracy drops while training accuracy rises, the model is likely overfitting.

#31 Data

Test Set

A held-out portion of the dataset used only once, after training is complete, to provide a final unbiased evaluation of the model's performance. It simulates how the model will perform on completely unseen real-world data. The test set should never be used for any training or hyperparameter decisions.

#32 Evaluation

Overfitting

A phenomenon where a model learns the training data too well — including its noise and outliers — and fails to generalise to new data. Symptoms include very high training accuracy but significantly lower validation/test accuracy. Common remedies include regularisation, dropout, data augmentation, and early stopping.

#33 Evaluation

Underfitting

A situation where the model is too simple to capture the underlying patterns in the data, resulting in poor performance on both training and test sets. It often occurs when the network architecture is too small, training is stopped too early, or the learning rate is too high. Solutions include increasing model capacity and training longer.

#34 Regularization

Regularization

A collection of techniques used to prevent overfitting by discouraging overly complex models. Common methods include L1 and L2 penalties that add a cost proportional to the magnitude of the weights, as well as dropout and data augmentation. Regularisation helps the model learn generalisable patterns rather than memorising the training data.

#35 Regularization

Dropout

A regularisation technique that randomly deactivates a fraction of neurons during each training step, forcing the network to avoid relying on any single neuron. This creates an implicit ensemble of many sub-networks, which improves generalisation. At test time, all neurons are active but their outputs are scaled accordingly.

#36 Architecture

Dense Layer (Fully Connected)

A layer in which every neuron is connected to every neuron in the previous layer, enabling the network to learn global combinations of features. In a CNN, dense layers typically appear at the end, after convolutional and pooling layers, to perform the final classification. They tend to have the most parameters in the network.

#37 Architecture

Flattening

The operation of converting a multi-dimensional feature map (e.g., a 7×7×64 tensor) into a one-dimensional vector so it can be fed into a dense layer. Flattening is the bridge between the spatial feature extraction of convolutional layers and the classification logic of fully connected layers. No learnable parameters are involved in this step.

#38 Evaluation

Accuracy

The most intuitive evaluation metric, representing the fraction of predictions the model got correct out of the total. While easy to understand, accuracy can be misleading on imbalanced datasets where one class dominates. In such cases, metrics like precision, recall, and F1-score provide a more complete picture.

#39 Evaluation

Error Rate

The complement of accuracy, measuring the proportion of incorrect predictions over all predictions (Error Rate = 1 − Accuracy). It provides a quick snapshot of how often the model makes mistakes. Like accuracy, it should be supplemented with other metrics when class distributions are uneven.

#40 Evaluation

Inference

The process of using a trained model to make predictions on new, unseen data. Unlike training, inference only involves a forward pass — no gradients are computed and no weights are updated. Optimising inference speed and efficiency is critical for deploying models in real-time applications.
